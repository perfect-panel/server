# V4.3 备份运维说明(决策 27)

## 1. binlog 启用

MySQL `my.cnf` / `docker-compose.yml` 的 mysql 服务里加:

```ini
[mysqld]
log_bin = /var/lib/mysql/mysql-bin
binlog_format = ROW
binlog_row_image = FULL
expire_logs_days = 14
sync_binlog = 1
```

容器启动后用 `SHOW VARIABLES LIKE 'log_bin'` 确认 `ON`。

## 2. 每日 dump

```bash
# crontab(每日 03:30)
30 3 * * * /usr/local/bin/ppanel-backup.sh >> /var/log/ppanel-backup.log 2>&1
```

`ppanel-backup.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail
TS=$(date +%Y%m%d-%H%M%S)
OUT_DIR=/backup/ppanel
mkdir -p "$OUT_DIR"
mysqldump --single-transaction --routines --triggers \
  -h "$MYSQL_HOST" -u "$MYSQL_USER" -p"$MYSQL_PASS" "$MYSQL_DB" \
  | gzip > "$OUT_DIR/$MYSQL_DB-$TS.sql.gz"
# 保留 14 天
find "$OUT_DIR" -type f -name '*.sql.gz' -mtime +14 -delete
# 推到异地(决策 27)
rclone copy "$OUT_DIR" "remote:ppanel-backup/$MYSQL_DB/" --include "*.sql.gz"
```

## 3. binlog 异地同步(可选,RPO 接近秒级)

启用 GTID 后用 `mysqlbinlog --read-from-remote-server` 持续抓取到异地存储:

```bash
mysqlbinlog --read-from-remote-server --raw \
  --host="$MYSQL_HOST" --user="$MYSQL_USER" --password="$MYSQL_PASS" \
  --result-file=/backup/binlog/ \
  $(mysql -h "$MYSQL_HOST" -u"$MYSQL_USER" -p"$MYSQL_PASS" -BNe "SHOW BINARY LOGS" | head -1 | awk '{print $1}')
```

## 4. 恢复演练 SLA

- **每月一次**:从最新 dump 恢复到隔离实例,跑 ppanel-server 启动检测确保 schema 兼容。
- 验收通过 = 从备份恢复后,`/v1/admin/server/list` 返回的 server 数与生产一致;`SELECT COUNT(*) FROM user_subscribe` 误差 < 0.5%。
