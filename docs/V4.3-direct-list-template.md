# V4.3 直连白名单模板用法(决策 39)

服务端在生成客户端订阅时已自动把 `server.direct_list`(per-node JSON 数组,管理端可改)
union 后注入到客户端模板上下文,变量名 `.DirectList`,管理员需在 **client subscribe template** 里
显式渲染规则。

## Clash / Stash / ClashMeta(YAML)

```yaml
proxy-groups:
  - name: PROXY
    type: select
    proxies:
      {{ range .Proxies }}- {{ .Name }}
      {{ end }}

rules:
{{ range directRulesClash .DirectList }}  - {{ . }}
{{ end }}
  - DOMAIN-SUFFIX,local,DIRECT
  - GEOIP,CN,DIRECT
  - MATCH,PROXY
```

或不用 helper,直接 range:

```yaml
rules:
{{ range .DirectList }}  - DOMAIN-SUFFIX,{{ . }},DIRECT
{{ end }}
  - MATCH,PROXY
```

## Surge / Loon / Shadowrocket(INI)

```ini
[Rule]
{{ range directRulesSurge .DirectList }}{{ . }}
{{ end }}
DOMAIN-SUFFIX,local,DIRECT
GEOIP,CN,DIRECT
FINAL,PROXY
```

## Quantumult X(自有语法)

```ini
[filter_local]
{{ range directRulesQuanX .DirectList }}{{ . }}
{{ end }}
host-suffix, local, direct
geoip, cn, direct
final, proxy
```

## 默认建议白名单(决策 39 上线 checklist)

每个节点的 `direct_list` JSON 数组建议至少包含:

- 面板域名 / 订阅域名(避免限速时锁死自我续费入口)
- `stripe.com`、`paypal.com`、`alipay.com`、`mch.weixin.qq.com`(支付通道)
- 客服 IM(如 Telegram、Discord)用到的域名

## 调试

订阅 URL 加 `?debug=1` 没有特殊效果(adapter 不解析此参数);可在管理端模板里加调试块:

```
# DEBUG direct list (移除上线前):
# {{ range .DirectList }}# {{ . }}
# {{ end }}
```

## 数据格式约束

`server.direct_list` 必须是合法 JSON array of string。空数组或 NULL 都 OK,模板 range 会自然不出行。
若解析失败会在 server 日志打 `[collectDirectList] bad json on server N`,不阻断订阅生成。
